## Naming Conventions

### Files

**Pattern: camelCase for sources, lowercase for config**
- Components: `FormStep.jsx`, `Landing.jsx`, `Dashboard.jsx` (PascalCase)
- Hooks: `useForm.js` (camelCase with `use` prefix)
- Data files: `questions.js` (camelCase)
- Config: `vite.config.js`, `tailwind.config.js`, `next.config.js` (kebab-case)

**Directories:**
- Feature dirs: `landing/`, `form/`, `roadmap/`, `segments/` (kebab-case)
- Functional dirs: `components/`, `hooks/`, `pages/`, `data/`, `lib/` (kebab-case, lowercase)

### Functions & Components

**Pattern: PascalCase for React components, camelCase for functions**
- Components: `export default function Landing() { ... }`
- Hooks: `export function useForm() { ... }`
- Handlers: `handleSubmit()`, `handleChange()`
- Utilities: `setAnswer()`, `toggleMulti()`, `canProceed()`

### Variables & State

**Pattern: camelCase**
- State: `const [currentStep, setCurrentStep] = useState(0)`
- Objects: `const answers = {}`
- Env vars: `VITE_SHEETS_WEBHOOK_URL`, `NEXT_PUBLIC_SUPABASE_URL` (UPPER_SNAKE_CASE)

### Types

**Pattern: PascalCase**
- Not heavily typed in analyzed code (mostly JSX/JS)
- TypeScript types would follow: `interface FormState { ... }`, `type Answer = string | string[]`

## Where to Add New Code

### New Landing Feature / Page

1. **Component file:** `apps/landing/src/[feature]/[ComponentName].jsx`
   - Example: `apps/landing/src/pricing/Pricing.jsx` for pricing page

2. **Add route in App.jsx:**
   ```jsx
   <Route path="/pricing" element={<Pricing />} />
   ```

3. **If it's a multi-step form:** Create `apps/landing/src/[feature]/hooks/use[Feature].js` for state management

4. **Styling:** Use Tailwind classes inline; define color tokens in `tailwind.config.js` if new colors needed

### New Main App Feature / Page

1. **Page file:** `apps/app/app/[feature]/page.jsx`
   - Example: `apps/app/app/settings/page.jsx`

2. **If auth-required:** Place in route group: `apps/app/app/(protected)/[feature]/page.jsx`

3. **Component for page:** `apps/app/app/components/[FeatureName].jsx` (if sharing across routes)

4. **Server state queries:** Use React Query hooks in client components, initialize in component or custom hook

5. **Database calls:** Use Supabase client from `apps/app/lib/supabase.js`

### Utilities & Shared Logic

**For Landing:**
- Shared utilities: `apps/landing/src/utils/` (create if needed)
- Shared constants: `apps/landing/src/constants/` (create if needed)

**For Main App:**
- Shared utilities: `apps/app/lib/utils.js` (or `lib/utils/` subdirectory)
- API helpers: `apps/app/lib/api.js` (Supabase client already here)

### Database Schema & Migrations

**Not analyzed.** When Supabase schema is needed:
- Migrations: Store in `apps/app/db/migrations/` (inferred structure)
- Schema docs: `apps/app/db/schema.md` (recommended)
