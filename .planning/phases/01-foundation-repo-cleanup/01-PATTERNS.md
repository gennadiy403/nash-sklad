**Pattern to apply** (RESEARCH §Pattern 1, lines 375-398):

```javascript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import { sentryVitePlugin } from '@sentry/vite-plugin'

export default defineConfig({
  plugins: [
    react(),
    // Must be LAST in plugins; only active in CI (auth token present)
    process.env.SENTRY_AUTH_TOKEN && sentryVitePlugin({
      org: 'oborotcrm',
      project: 'landing',
      authToken: process.env.SENTRY_AUTH_TOKEN,
    }),
  ].filter(Boolean),

  },
```

**Gotchas:**

- After CF Pages cutover, `preview.allowedHosts` becomes dead code (CF Pages doesn't use `vite preview`). Keep it — it's harmless and useful if someone runs preview locally.

**Pattern to apply** (RESEARCH §Pattern 1, lines 326-372 — adapted):

```javascript
import { StrictMode } from 'react'
import React from 'react'
import { createRoot } from 'react-dom/client'
import * as Sentry from '@sentry/react'
import {
  useLocation,
  useNavigationType,
  createRoutesFromChildren,
  matchRoutes,
} from 'react-router-dom'
import './index.css'
import App from './App.jsx'
import EnvErrorBanner from './components/EnvErrorBanner'

// Parse + validate env at module load (throws EnvError on missing required vars)
let env, envErrors
try {
  env = (await import('./lib/env')).env
} catch (e) {
  envErrors = e.errors || [e.message]
}

// Init Sentry only if env OK and DSN present
if (env?.VITE_SENTRY_DSN) {
  Sentry.init({
    dsn: env.VITE_SENTRY_DSN,
    integrations: [
      Sentry.browserTracingIntegration(),
      Sentry.reactRouterV7BrowserTracingIntegration({
        useEffect: React.useEffect,
        useLocation,
        useNavigationType,
        createRoutesFromChildren,
        matchRoutes,
      }),
      Sentry.replayIntegration({ maskAllText: false, blockAllMedia: false }),
    ],
    tracesSampleRate: 0.1,
    replaysSessionSampleRate: 0.0,
    replaysOnErrorSampleRate: 1.0,
    environment: import.meta.env.MODE,
    release: env.VITE_SENTRY_RELEASE || undefined,
  })
}

createRoot(document.getElementById('root')).render(
  <StrictMode>
    {envErrors ? <EnvErrorBanner errors={envErrors} /> : <App />}
  </StrictMode>,
)
```

**Gotchas:**
- RESEARCH's snippet uses top-level `env` proxy access; simpler: wrap env import in try/catch at module-load.
- `maskAllText: false` is intentional for waitlist-stage product (RESEARCH §Pitfall 8).
- `reactRouterV7BrowserTracingIntegration` exists in `@sentry/react@10.49.0` (RESEARCH §Assumption A1 — verify at install).
